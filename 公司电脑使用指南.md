# IPO Scoring Skill — 公司电脑快速上手指南

> 本文档用于在公司电脑上快速配置和使用 IPO 评分 skill。

---

## 一、首次配置（只需做一次）

### 1. Clone 项目仓库

```bash
git clone https://github.com/raymondzhang/ipo-margin-eval.git
cd ipo-margin-eval
```

### 2. 验证 Skill 已加载

打开 Kimi Code CLI（或 VS Code 中的 Kimi 插件），直接输入：

```
帮我看下这个项目里有哪些 skill
```

如果看到 `ipo-scoring`、`ipo-backtest`、`ipo-funding-application` 被列出，说明配置成功 ✅

也可以直接测试触发：

```
评估这份招股书
```

## 二、Skill 总览

| Skill | 用途 | 触发关键词 |
|-------|------|-----------|
| `ipo-scoring` | 对新股招股书进行7维度评分，生成双语Word报告 | "评估这份招股书" |
| `ipo-backtest` | 回溯验证评分模型对历史IPO的预测有效性 | "回溯验证评分" |
| `ipo-funding-application` | 基于评分结果生成内部融资额度申请邮件 | "写孖展申请邮件" |

## 三、日常使用流程

### 场景 A：评估新 IPO（ipo-scoring）

每次评估新 IPO 时，按以下步骤操作：

#### Step 1: 准备招股书 PDF

将招股书 PDF 文件放入项目目录，例如：

```
~/Downloads/2026XXXXXX_prospectus.pdf
```

#### Step 2: 触发 Kimi 评估

在 Kimi 对话中输入（以下任意一种均可触发）：

```
评估这份招股书
```

```
对这个IPO做孖展评分
```

```
按IPO Score Guidelines给这个新股打分
```

#### Step 3: 提供破发率数据（关键！）

Kimi 会**优先询问**你：

> "请提供[YYYY]年Q[X]港股新股首日破发率（%），或告知我应采用的数值（如'低于10%'）。"

**你有两种方式回答：**

| 方式 | 示例 | 说明 |
|------|------|------|
| 直接给数字 | "9.5%" | 最精确，推荐 |
| 给定性描述 | "低于10%" | Kimi 会自动映射到对应分数档 |

> 💡 **Tip**：如果你公司有 Wind 终端，直接查当季数据最准。没有的话，看当天财经媒体（财联社、新浪财经）的季度 IPO 复盘报道也可以。

#### Step 4: 等待报告生成

Kimi 会自动完成：
1. 读取 PDF 提取关键信息
2. 按 7 个维度评分
3. 自动获取恒生指数月度涨跌幅
4. 生成分类和孖展建议
5. 输出 **简体中文 + 繁體中文双语 Word 报告**

#### Step 5: 查看报告

报告默认保存在项目目录：

```
ipo-margin-eval/
└── IPO_Score_Report_XXXX.docx
```

用 Microsoft Word 或 WPS 打开即可。

### 场景 B：回溯验证评分模型（ipo-backtest）

用于验证评分模型对历史 IPO 的预测是否准确。

#### 快速触发

```
帮我回溯一下2026年4月上市的IPO，验证评分模型有效性
```

#### 半自动工作流

1. **Kimi 搜索数据**：自动搜索 etnet、东方财富等网站获取历史 IPO 表现数据
2. **人工确认**：用户核对数据准确性
3. **生成报告**：脚本自动生成 Excel 统计报告

#### 手动执行（如需自定义）

```bash
cd .agents/skills/ipo-backtest/scripts

# 获取IPO数据（内置2026年4月默认数据集）
python fetch_ipo_data.py --output ipo_data.json

# 准备评分数据（用户手动创建 scores.json）
# 格式：{"06656": {"total_score": 95, "base_score": 85, "classification": "白名单"}, ...}

# 生成回溯报告
python generate_backtest_report.py --input ipo_data.json --scores scores.json --output backtest_report_202604.xlsx
```

#### 报告内容

- **Sheet 1 原始数据**：每只新股的评分 + 实际表现
- **Sheet 2 统计汇总**：首日破发率、平均涨幅、评分有效性验证
- **Sheet 3 分组分析**：高分组/中分组/低分组的收益对比

#### 持续监控

对已评估但未上市的新股（如可孚1187、天星1609），上市后可手动添加到监控日志，对比事前评分与实际表现。

### 场景 C：生成融资申请邮件（ipo-funding-application）

基于评分结果，向管理层申请孖展融资额度审批。

#### 快速触发

```
帮我写一份天星医疗的孖展融资申请邮件
```

#### 工作流

1. **准备 JSON 数据**：包含股票信息、融资参数、评分结果
2. **生成邮件**：脚本自动生成 Markdown（+ 可选 Word）
3. **人工审核**：确认额度一致性、利率合理性
4. **发送**：复制到邮件客户端或作为附件发送

#### 手动执行

```bash
cd .agents/skills/ipo-funding-application/scripts

# 创建 data.json（参考 references/email-template.md 的变量说明）
# 然后生成邮件：

# Markdown only
python generate_email.py --input data.json --output IPO_Funding_App_1609.md --format md

# Markdown + Word
python generate_email.py --input data.json --output IPO_Funding_App_1609 --format both
```

#### 邮件结构

- **Part 1 — 致 Celine**：审阅请求 + 竞品券商利率参考
- **Part 2 — 致管理层**：公司及市场简介 + 孖展融资安排 + 港交所公告链接
- **Part 3（可选）— 评分参考**：7维度评分汇总 + IPO分类 + 孖展建议

## 四、评分维度速查

| 维度 | 满分 | 外部数据？ |
|------|------|-----------|
| 过往3年盈利情况 | 30 | ❌ 招股书内 |
| 分配机制 | 10 | ❌ 招股书内 |
| 基石投资人 | 15 | ❌ 招股书内 |
| 定价情况（静态PE） | 20 | ❌ 招股书内 |
| 稳价机制 | 10（奖励） | ❌ 招股书内 |
| **过往一季度新股首日破发率** | **15** | **✅ 需要你提供** |
| **恒生指数近1个月涨跌幅** | **10** | **✅ Kimi自动获取** |

**总分：110 分（基础分 100 + 奖励分 10）**

## 五、白名单 / 灰名单 / 黑名单速查

| 分类 | 标准 | 决策 |
|------|------|------|
| **白名单** | 基础分 ≥ 80/100 | 可直接开放打新融资额度 |
| **灰名单** | 18A / 18C 上市企业 | 按内部政策处理 |
| **黑名单** | 触发以下任一条件：(1) 基石≤3家或≤20%；(2) 静态PE超行业龙头20%+；(3) 重大负面舆情 | 不建议开放，或要求客户增加保证金 |

## 七、常见问题

### Q1: Kimi 说找不到 skill？

确认项目目录下有 `.agents/skills/ipo-scoring/SKILL.md` 等文件。如果没有，重新 clone：

```bash
git clone https://github.com/raymondzhang/ipo-margin-eval.git
```

### Q2: 破发率数据源口径不一致怎么办？

Kimi 会搜索多个来源。如果来源冲突超过 5 个百分点，会呈现范围让你确认。**优先采用你公司内部统计的数据**（如 Wind）。

### Q3: 报告里的繁体中文版可以去掉吗？

可以。在触发 Kimi 时加一句：

```
评估这份招股书，只需要简体中文版
```

### Q4: 评分标准要调整怎么办？

联系 skill 维护者（你在家里的电脑）修改 `SKILL.md` 和 `references/scoring-rubric.md`，然后 push 到 GitHub，公司电脑 `git pull` 即可同步更新。

### Q5: 回溯报告缺少评分数据怎么办？

`ipo-backtest` 支持仅生成表现统计（不含评分对比）。如需评分对比，请手动创建 `scores.json` 文件，格式参考 `ipo-backtest/SKILL.md` 中的 Input Format 章节。

### Q6: 融资申请邮件的额度不一致怎么办？

脚本会自动在邮件中标注 ⚠️ 注意 项，提示用户确认预批额度与致Celine的总额是否一致。**请务必人工核对后再发送**。
